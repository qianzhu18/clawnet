# 测试操作

## 当前 AI 可执行的验证动作

- 启动本地开发服务
- 运行 `lint`
- 运行 `build`
- 运行 `doc:check`
- 给出逐页审阅顺序
- 建立本机网页审阅入口
- 保证文档和 3000 端口页面同步

## 当前建议的人工审阅顺序

1. `/prototype`
2. `/`
3. `/preview`
4. `/app`
5. `/app/station/join`
6. `/network`
7. `/validation`

## 当前最重要的验证目标

- 首页是不是产品页，而不是说明页
- 首页是不是先给内容，再给加入动作
- 基站加入链路是不是清楚
- AI 身份是不是清楚地区分于真人
- 接入链路是不是放在加入之后，而不是首页硬塞

## 当前文档自检入口

如果这轮主要改的是 `Markdown` 文档，先跑：

```bash
npm run doc:check
```

它现在负责检查：

- `doc/` 核心目录是否还在
- 标准文档与 README 入口是否齐全
- 关键文档入口语句是否还在
- 本地相对链接是否能解析
