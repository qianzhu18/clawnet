# 编码操作

## 当前 AI 可执行的原型编码动作

- 调整页面结构
- 补齐 mock 数据
- 调整页面语言为对外产品语气
- 改善移动端层级
- 调整右下动作入口
- 增加原型审阅页
- 修复跳转链路
- 把本机预览地址同步给用户
- 把改动提交到 git 保持可追溯

## 当前编码边界

当前只服务：

- `概要设计`
- `产品原型设计`

当前不主动展开：

- 后端 API
- 数据库建模
- 企业后台
- 协议层细节
- 复杂多 agent 调度

## 编码目标

不是先把系统做完。

而是：

- 让原型看起来像产品
- 让链路能在本地浏览器里走通
- 让你可以亲自审核每一个交互

## 编码交付硬规则

- 每次代码修改后，都要给出本机预览地址，默认至少包含：
  - `http://127.0.0.1:3000/`
  - `http://127.0.0.1:3000/prototype`
  - `http://127.0.0.1:3000/validation`
- 只要产生了有效代码改动，就要同步到 git，至少做到：
  - 当前分支可见改动
  - 有清晰 commit 信息
  - 改动来源可追溯
- 面向用户可见的页面文案，不允许出现“当前原型 / 应该 / 这页负责 / 先看再决定 / 当前版本”这类内部开发说明语气
- 交互逻辑说明、设计判断、开发约束，只能进入 `design/` 或 `commands/`，不能直接暴露到产品页面
