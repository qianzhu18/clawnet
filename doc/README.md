# 文档系统说明

当前公开文档系统只保留 4 个入口：

- `commands/`
- `design/`
- `history/`
- `fixes/`

当前文档按软件工程顺序公开呈现：

1. `需求分析`
2. `概要设计`
3. `产品原型设计`
4. `详细设计`
5. `开发`
6. `测试`
7. `迭代`

当前真正活跃的阶段仍然是：

- `需求分析`
- `概要设计`
- `产品原型设计`

如果后续要把少量文档单独发给外部设计师，不要直接发整套 `design/`。

请只整理并发送：

- `design/设计交付包/`

## 当前阅读顺序

如果你是第一次进入项目，按这个顺序读：

1. `../AGENTS.md`
2. `design/标准文档/1-需求分析.md`
3. `design/标准文档/2-概要设计.md`
4. `design/标准文档/3-产品原型设计.md`
5. `design/标准文档/4-详细设计.md`
6. `design/标准文档/5-开发.md`
7. `design/标准文档/6-测试.md`
8. `design/标准文档/7-迭代.md`
9. `design/原型细节/README.md`
10. 浏览器打开 `http://127.0.0.1:3000/prototype`

   - `/`

## 各目录职责

### `commands/`

给协作者和 AI 助手看的“可执行动作说明”。

内容包括：

- 可以让 AI 做哪些文档整理
- 可以让 AI 做哪些原型整理
- 本机怎么验证
- 当前协作的附加要求

### `design/`

当前项目的公开设计文档主区。

分成 4 层：

- `标准文档/`
  - 软件工程主线文档
- `概要细节/`
  - 对概要设计的专题补充
- `原型细节/`
  - 对原型设计的逐页补充
- `详细设计/`
  - 先冻结界面设计文档规则，再进入真实页面详细设计

### `history/`

旧文档、旧目录系统、过渡稿、历史记录都放这里。

如果你只是要理解当前项目，不要从这里开始。

### `fixes/`

当前不作为主工作台使用。

后续如果进入问题修复和回归记录阶段，再正式展开。

## 当前管理原则

1. 对外公开优先看 `design/标准文档/`
2. 当前阶段不再并列维护大量同级主文档
3. 原型详细稿可以多，但入口只能有一份
4. 软件工程 `4-7` 阶段先建立规则，再按阶段展开
5. 老材料不删除，只归入 `history/`
6. 本机网页审阅入口固定为 `/prototype`

## 当前一句话总结

如果你现在想审核项目，只做两件事：

1. 先看 `design/标准文档/1-2-3`
2. 再去 `localhost:3000/prototype` 按顺序看网页

如果你现在想准备后续界面设计稿，只做两件事：

1. 先看 `design/标准文档/4-详细设计.md`
2. 再看 `design/详细设计/` 下的 3 份规则文档

如果你现在想整理 `Markdown` 工作台，只做三件事：

1. 先看 `commands/Markdown工作台.md`
2. 再跑 `npm run doc:check`
3. 再把结论写回对应标准文档或 `history/`
