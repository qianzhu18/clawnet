# 文档系统说明

当前公开文档系统只保留 4 个入口：

- `commands/`
- `design/`
- `history/`
- `fixes/`

当前阶段只看两步：

1. `概要设计`
2. `产品原型设计`

也就是说，现在不要先跳到详细设计、开发实现或历史归档里。

## 当前阅读顺序

如果你是第一次进入项目，按这个顺序读：

1. `../AGENTS.md`
2. `design/标准文档/1-需求分析.md`
3. `design/标准文档/2-概要设计.md`
4. `design/标准文档/3-产品原型设计.md`
5. `design/原型细节/原型审阅指南.md`
6. 浏览器打开 `http://127.0.0.1:3000/prototype`
7. 再按顺序打开：
   - `/`
   - `/preview`
   - `/app`
   - `/app/station/join`
   - `/network`
   - `/validation`

## 各目录职责

### `commands/`

给协作者和 AI 助手看的“可执行动作说明”。

内容包括：

- 可以让 AI 做哪些文档整理
- 可以让 AI 做哪些原型整理
- 本机怎么验证
- 当前协作的附加要求

### `design/`

当前项目的公开设计文档主区。

分成 4 层：

- `标准文档/`
  - 软件工程主线文档
- `概要细节/`
  - 对概要设计的专题补充
- `原型细节/`
  - 对原型设计的逐页补充
- `详细设计/`
  - 未来进入详细设计时再展开

### `history/`

旧文档、旧目录系统、过渡稿、历史记录都放这里。

如果你只是要理解当前项目，不要从这里开始。

### `fixes/`

当前不作为主工作台使用。

后续如果进入问题修复和回归记录阶段，再正式展开。

1. 对外公开优先看 `design/标准文档/`
2. 当前阶段不再并列维护大量同级原型文档
3. 原型详细稿可以多，但入口只能有一份
