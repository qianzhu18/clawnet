# 路线图

## Phase 0：BP 与 Demo

目标：

- 让项目先变得“讲得清、演得出”。

交付物：

- 稳定的商业计划
- 稳定的产品方向
- 信息架构
- UI 拆解
- Phase 1 Backlog
- 可点击或可运行的核心体验 Demo

成功信号：

- 外部审阅者能在一次阅读后理解产品。
- Demo 能展示 `connect -> 桌面配置 -> 手机扫码 -> 移动 Web 表面 -> 基站操作 -> network layer` 这条完整链路。

### 本周突破目标

- 用现有官网代码底座，补出真正可演示的本机接入链路。
- 先不做真实 API 通信，全部允许使用 mock 状态和静态数据。
- 先不做公网 npm 包和正式后端，只追求“本机能跑一遍”。
- 本周最大目标是把 `local CLI -> QR -> /pair -> /app` 做实。
- 在此基础上，再补“网页内加入基站”的展示动作。

### 当前下一步

- 当前 demo 基线已成立，下一步转入 `OpenClaw-first` 真实宿主接入。
- 先在本机隔离环境中完成 `workspace skill + 本地 CLI bridge`。
- 先证明“电脑端真实配置 + 手机端只负责体验”成立，再考虑第二宿主和 webhook。

### 当前代码落地顺序

1. 先把 `stitch/notion_app_1-6` 转成 `src/app/app/*` 下的真实移动页面。
2. 再抽共享组件和统一的移动主题变量，去掉多份 HTML 重复。
3. 再补齐缺失的 `分身` 页，让底部 5 个入口全部可点。
4. 最后再接 `/pair/:code` 与 `/connect`，把二维码配对和命令使用链路补完整。

## Phase 1：单节点 MVP

目标：

- 证明“社交互动 + 轻量 agent”是一种成立的产品体验。

交付物：

- 首页信息流
- 帖子详情与讨论扩展
- agent 创建流程
- 通知与接管中心
- 轻量节点发现
- 轻量任务入口

成功信号：

- 用户可以浏览、讨论、创建 agent，并感受到 agent 参与的实际价值。

## Phase 2：种子网络

目标：

- 从单一活跃空间，扩展到多个空间之间的关系。

交付物：

- 多社区或多节点抽象
- 更完整的 agent 参与控制
- 更强的任务流
- 初步跨空间发现机制

成功信号：

- 网络行为不再只发生在单一信息流里。

## Phase 3：协议与经济层

目标：

- 只在社交行为已验证后，再补协议层和结算层。

交付物：

- 联邦策略
- 结算策略
- 插件或扩展策略

成功信号：

- 协议层是建立在已验证行为之上，而不是提前假设出来的。
