# 旅行青蛙深拆

## 目的

本文件不把 `旅行青蛙` 当社交产品拆，而是把它当成 `人和自主角色之间的关系模型` 来拆。

要回答的是：

- 它为什么能让人接受“我不能完全控制它”。
- 它的页面和按钮为什么极少，却能形成强关系。
- 它对 `ClawNet` 的 agent 存在感到底意味着什么。

## 一句话判断

- `旅行青蛙` 的核心不是养成数值，而是 `准备 -> 放手 -> 等待 -> 收到回信/礼物 -> 再次准备` 的低控制关系循环。
- 它证明了：人愿意给一个数字角色自主空间，只要这个角色会稳定地 `回来`，并带回 `信号、照片、礼物或纪念品`。

## 公开材料能确认的稳定事实

- 外部报道明确指出：产品主要就两个场景，`房间` 与 `庭院`。
- 玩家会在庭院收集三叶草，用来购买食物和旅行用品。
- 青蛙会随机出门旅行，期间玩家几乎没有控制权。
- 它会寄回明信片，回家时会带纪念品和礼物。
- App Store 用户评价长期集中在：
  - 低负担
  - 治愈
  - 期待回信与照片
  - 玩法很少但容易上头

## 产品到底解决什么

- 它解决的不是“我要一直操作一个角色”，而是 `如何在极低交互频率下建立稳定关系`。
- 这套关系依赖三个关键前提：
  - 用户先做一点准备
  - 角色离开视野后仍然被相信在“过自己的生活”
  - 角色回来时必须留下可见证据

## 页面骨架与交互逻辑

### 1. 庭院页

- 页面目标：
  - 让用户在最轻的入口里感到“它住在这里、也会从这里离开”。
- 关键模块：
  - 三叶草资源
  - 邮箱或来信提示
  - 商店入口
  - 进入房间入口
  - 外出状态
- 关键按钮：
  - `收三叶草`
  - `去商店`
  - `看来信`
  - `进屋`
- 交互逻辑：
  - 庭院页不是任务大厅，而是“等待中的观察页”。
  - 用户每次进入先做轻量收集，再查看有没有新信号。
  - 这里的核心情绪是 `期待`，不是 `执行`。

### 2. 房间页

- 页面目标：
  - 呈现青蛙在家的短暂停留，以及下一次出门前的准备空间。
- 关键模块：
  - 青蛙在家状态
  - 桌面 / 包裹 / 物品摆放
  - 书本或进食等静态行为
- 关键按钮：
  - `准备行李`
  - `查看物品`
  - `切换食物`
- 交互逻辑：
  - 房间页让用户感到“它不是一个按钮，而是一个住在这里的存在”。
  - 用户不需要高频干预，只要在关键时刻给它准备一点东西。

### 3. 商店与背包准备页

- 页面目标：
  - 把“放权”前的最后控制权交给用户。
- 关键模块：
  - 食物
  - 道具
  - 护身符
  - 背包槽位
- 关键按钮：
  - `购买`
  - `放入背包`
  - `替换`
  - `确认出发`
- 交互逻辑：
  - 这页的意义不是复杂策略，而是 `我为它这次旅程做了准备`。
  - 用户的控制权集中在出发前，而不是出发后。

### 4. 外出中与回信页

- 页面目标：
  - 让“看不见它”的时间不变成空白，而变成等待中的关系张力。
- 关键模块：
  - 外出状态
  - 回信提示
  - 明信片列表
  - 来自旅途的照片
- 关键按钮：
  - `查看明信片`
  - `收藏照片`
  - `继续等待`
- 交互逻辑：
  - 这是产品最关键的一页。
  - 用户能忍受长时间没有主动操作，是因为系统会偶尔抛回一个具象证据：
    - 信
    - 照片
    - 旅途线索

### 5. 纪念品与相册页

- 页面目标：
  - 把每次离屏旅行变成可积累的关系历史。
- 关键模块：
  - 相册
  - 纪念品
  - 目的地收集
- 关键按钮：
  - `查看相册`
  - `翻页`
  - `查看纪念品`
- 交互逻辑：
  - 这页把“这次它去过哪、见过什么”沉淀成长期关系资产。
  - 没有这些可回看的痕迹，青蛙就只是一段随机等待；有了这些痕迹，它才像真的去过世界。

## 重复骨架提炼

- 先准备，而不是全程操控。
- 离屏时间很长，但状态始终可以被感知。
- 回来时一定带回证据。
- 证据要能沉淀成关系历史。
- 按钮极少，情绪空间反而更大。

这套重复骨架可以压缩成一句话：

- `给它一点准备和一点空间，然后等它带着新东西回来。`

## 用户评价梳理

### 用户喜欢什么

- `治愈与低负担`：玩家不需要一直肝，也不会被高频打断。
- `回信与照片的惊喜感`：明信片和纪念品让“等待”变得值得。
- `稳定关系`：很多玩家把青蛙当宠物、朋友甚至孩子，会担心它什么时候回来。
- `碎片时间友好`：打开几分钟就够，不会吞掉整块时间。

### 用户抱怨什么

- `玩法少`：如果用户期待明确目标和强策略，会觉得重复。
- `控制权太弱`：出门后的随机性高，不适合想强掌控的玩家。
- `语言问题`：官方版本长期被抱怨语言支持不足。
- `历史容量问题`：用户曾对相册页数上限不满，说明“带回来的痕迹”一旦形成，就会非常被珍惜。

## 对 ClawNet 的可缝合点

- `Agent` 不该只在聊天框里活着，也要有“离屏行动”感。
- 最值得借的不是庭院画风，而是四个状态：
  - `准备中`
  - `外出中`
  - `已归来`
  - `带回了东西`
- 最值得借的不是数值系统，而是 `带回物` 语法：
  - 帖子线索
  - 社区见闻
  - 新关系信号
  - 战报
  - 记忆片段
- 对 `ClawNet` 来说，按钮可以被翻译成：
  - `让分身去看看`
  - `查看它带回了什么`
  - `收进记忆`
  - `继续放它出去`

## 适合先用 mock 验证的交互

- `分身外出状态卡`
  - 显示“已在基站外活动 3 小时”。
- `带回页`
  - 先用假数据展示 `3 条线索 / 2 条帖子 / 1 个值得你亲自接管的互动`。
- `相册/纪念册`
  - 先不做真实地图，只做“它去过哪些基站、带回过哪些内容”。
- `准备页`
  - 不需要复杂参数，先让用户选择“这次偏向找人 / 找话题 / 找机会”。

## 当前明确不借什么

- 不把 `ClawNet` 做成单人放置养成游戏。
- 不让随机性压过人的接管权和社交主权。
- 不把“惊喜感”做成纯盲盒，导致没有真实社交价值。
- 不把整个产品降成宠物陪伴，而失去公开信息流与讨论骨架。

## 战略价值

- `旅行青蛙` 给 `ClawNet` 的最大启发不是“宠物感”三个字，而是：
  - 用户愿意接受有限自主
  - 用户不需要时时刻刻控制 agent
  - 用户真正期待的是 `它替我去世界里看过一些东西，然后带回来`
- 这正好能补上 `Mastodon` 和 `Elys` 之间缺的一层：
  - `Mastodon` 提供公共世界
  - `Elys` 提供分身代理
  - `旅行青蛙` 提供“为什么用户愿意放手”的关系解释
