# 开发流程

## 目标

用当前真实阶段驱动推进，而不是从想法直接跳到代码。

## 当前 active 顺序

当前 active 区只围绕这三层展开：

1. `当前边界与目标`
2. `概要分析`
3. `产品原型设计`

`03推进记录/` 只负责管理范围、决策和本周待办。

## 阶段 1：当前边界与目标

主文件：

- `01概要分析/当前边界与目标.md`

动作：

- 收敛目标用户
- 收敛当前范围
- 收敛当前不做什么
- 收敛当前主链路

退出条件：

- 读者能在一份文件里理解当前产品要做什么

## 阶段 2：概要分析

主文件：

- `01概要分析/概要分析.md`
- `01概要分析/OpenClaw融入微播客交互.md`

动作：

- 定系统分层
- 定对象边界
- 定核心状态
- 定宿主如何进入微播客产品形态

退出条件：

- 已能回答“当前用什么方法实现”
- 已知道原型该从哪几页开始

## 阶段 3：产品原型设计

主文件：

- `02产品原型/产品原型设计.md`
- `02产品原型/主线页面交互骨架.md`

动作：

- 画关键页面
- 冻结重复骨架
- 明确每页模块和按钮
- 明确状态稿

退出条件：

- 读者能照着文档开始出低保真或高保真原型

## 当前进入实现前的顺序

如果还没准备进入实现，应先按下面顺序推进：

1. 冻结 `当前边界与目标`
2. 冻结 `概要分析`
3. 冻结 `产品原型设计`
4. 冻结 `主线页面交互骨架`
5. 再决定是否把 `详细设计` 从 `99备份/非当前阶段/` 拉回 active

## 推进记录

当前目录：

- `03推进记录/`

职责：

- 管当前范围
- 管关键取舍
- 管本周任务

## Ready 的定义

一个功能只有满足以下条件，才算 ready：

- 边界文档里能找到它的来源
- 概要分析已说明它位于哪一层
- 原型设计已说明页面和模块
- 它已进入 `03推进记录/范围清单.md`
- 重复骨架和按钮语法已冻结

## Done 的定义

一个功能只有满足以下条件，才算 done：

- 验收标准满足
- 关键取舍已写进 `03推进记录/决策日志.md`
- 后续工作已写入 `03推进记录/范围清单.md` 或下轮文档

## 反模式

避免以下情况：

- 没有冻结边界和骨架就先写代码
- 从 `99备份/` 开始读项目
- 把边界、概要分析、原型、详细设计混在一个文件里
- 让关键结论只存在于聊天记录里
