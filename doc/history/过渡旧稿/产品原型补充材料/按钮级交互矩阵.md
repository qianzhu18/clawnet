# 按钮级交互矩阵

## 1. 文档定位

这份文档只做一件事：

- 把当前真实原型的每一个页面、每一个真实按钮，拆到可直接做网页验证的粒度

它不替代：

- `产品原型设计.md`
- `逐页详细设计稿.md`
- `页面按钮与交互结果清单.md`

它补的是最细一层：

- 这个按钮在哪
- 点击前提是什么
- 点击后去哪
- 页面会出现什么结果
- 在网页里应该怎么验证

网页验证入口固定为：

- `/validation`

## 2. 使用规则

- 只记录当前代码里真实存在的按钮和输入控件
- 不把候选功能写成现状
- 如果某页当前没有独立按钮，要明确写“无独立按钮”
- 页头品牌区、右上分身入口、右下动作按钮，单独作为全局控件核对，不在每个一级页重复展开
