# 页面按钮与交互结果清单

## 1. 文档定位

这份文件只做一件事：

- 把当前真实原型里“能点的东西”全部列出来

如果你要看最细粒度的版本，直接继续看：

- `产品原型/按钮级交互矩阵.md`

使用方法：

- 看某一页时，不再只问“这页叫什么”
- 而是直接看：
  - 有哪些按钮
  - 点击后去哪里
  - 应该看到什么结果

网页验证入口：

- `/validation`

如果你不想对着文档逐项比对，可以直接在网页验证板里勾选。

## 5. 快速可视化验收重点

| 页面 | 最少要看到什么，才算页面成立 |
| --- | --- |
| `/` | 三条入口都明确，且 `/pair` 没被放成首页入口 |
| `/preview` | 帖子流真实可点，不是静态海报 |
| `/posts/:id` | `@agent`、建议审批、任务草案都能直接操作 |
| `/agents/new` | 三问能走完，并进入完成态 |
