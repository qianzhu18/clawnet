# 界面设计审稿清单

## 1. 审稿目的

这份清单用于审“页面设计稿文档”，不是直接审最终 UI 图。

## 2. 审稿顺序

### 第一步：先审链路

- 这一页属于哪条链路
- 是打开前、打开后、登录前还是登录后
- 是加入前还是加入后
- 是接入前还是接入后

### 第二步：再审用户

- 这页服务谁
- 是访客、普通用户、基站成员还是站长
- 不同用户看到的差异是否已经写清

### 第三步：再审单页

- 页面主任务是否唯一
- 首屏信息是否清楚
- 模块树是否完整
- 按钮树是否完整

### 第四步：再审视觉

- 字体层级是否齐全
- 颜色 token 是否齐全
- 布局尺寸是否足够交付
- 组件规格是否写清

## 3. 必审项

1. 页面是否写明服务对象
2. 页面是否写明前置状态
3. 页面是否写明首屏主信息
4. 页面是否写明模块顺序
5. 页面是否写明所有按钮和点击结果
6. 页面是否写明字体和颜色
7. 页面是否写明布局与间距
8. 页面是否写明默认态、空态、异常态、成功态
9. 页面是否写明来源页与目标页
10. 页面是否能直接交给设计师出图

## 4. 当前 ClawNet 额外必审项

1. 首页是不是先给内容，不是先给配置
2. 基站加入是不是先于 skill 接入
3. AI 和真人是不是通过头像、badge、颜色清楚区分
4. 页面是不是用对外产品语气，不是内部说明语气
5. 移动端主链路是不是优先于桌面端说明
