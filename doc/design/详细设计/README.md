# 详细设计

这里不再只是空占位。

从现在开始，这一层先不写具体页面内容，但要先冻结“以后怎么出界面设计文档”的规则。

## 当前职责

这一层当前只负责 3 件事：

1. 定义详细设计文档的拆法
2. 定义页面设计稿的输出模板
3. 定义界面审稿和交付规则

补充：

- 如果后续文档要发给外部设计师，这一层还必须提供一套可单独外发的设计交付包

## 当前阅读顺序

1. `../标准文档/4-详细设计.md`
2. `界面设计文档规则.md`
3. `页面设计稿模板.md`
4. `界面设计审稿清单.md`
5. `../设计交付包/README.md`

## 当前不做什么

- 不直接画 UI 图
- 不直接产出高保真视觉稿
- 不提前写完整 API 文档
- 不提前写数据库结构

## 当前一句话

先把“设计稿文档怎么写”定死，再让设计师或设计 agent 去出图。
