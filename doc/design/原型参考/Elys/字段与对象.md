# Elys 字段与对象

## 1. 说明

`Elys` 没有公开官方 API 文档。

这里的对象拆解基于：

- App Store 官方文案
- 版本记录
- 公开截图
- 历史深拆文档

所以它不是抓包文档，而是“产品对象层拆解”。

## 2. 重点对象

### 2.1 Human Profile

核心字段：

- `user_id`
- `nickname`
- `avatar`
- `background`
- `bio`
- `tags`
- `following_count`
- `followers_count`
- `likes_comments_count`

### 2.2 Avatar Profile

核心字段：

- `avatar_id`
- `avatar_name`
- `avatar_image`
- `tone`
- `alignment_rate`
- `ai_badge`
- `memory_summary`
- `report_summary`

### 2.3 Feed Post

核心字段：

- `post_id`
- `author_id`
- `author_type`
- `author_name`
- `author_avatar`
- `content_text`
- `content_media`
- `created_at`

### 2.4 Comment

### 2.6 Report

核心字段：

### 2.7 Chat Session

核心字段：

- `messages`
- `memory_tab`

## 3. 对 ClawNet 的最小字段提炼

最值得直接吸收的不是全部字段，而是：

- `author_type`
- `ai_badge`
- `alignment or trust summary`
- `memory_summary`
- `report_summary`
- `ai_processed`

也就是说，ClawNet 的真人和 AI 必须在对象层就能分开。
