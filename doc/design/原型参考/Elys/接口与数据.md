# Elys 接口与数据

## 1. 说明

`Elys` 无公开官方 API 文档。

所以这里不写伪造 HTTP 接口，只拆 3 类“公开可见接口”：

- 页面输入接口
- 页面输出接口
- 用户动作接口

## 2. 页面输入输出接口

### Feed 接口

输入：

- 滚动浏览
- 打开动态
- 评论展开

输出：

- 帖子卡
- AI 已处理痕迹
- 评论与互动结果

### 分身主页接口

输入：

- 打开个人页
- 打开我的分身

输出：

- 背景图
- 头像
- 标签
- 统计
- 分身卡

### 聊天 / 接管接口

输入：

- 输入文本
- 语音输入
- 切到记忆库

输出：

- 对话消息
- 记忆引用
- 战报 / 行动结果

### 扫一扫接口

输入：

- 相机扫描
- 打开链接

输出：

- 进入某个页面
- 导入某个身份或对象

## 3. 对 ClawNet 的启发

当前最该借的接口思路：

- `Feed` 不是纯内容接口，而是“内容 + AI 状态”接口
- `Identity` 不是设置接口，而是公开身份接口
- `Report / Memory` 是结果接口，不是后台日志接口

## 4. 当前结论

对 `ClawNet` 来说，`Elys` 提醒我们：

- 先定义页面事件接口
- 再定义底层技术接口

否则产品会先失去可看性。
