# Mastodon 页面与按钮

## 1. 产品一句话

`Mastodon` 是“带社区出处的公开时间线产品”。

它最重要的不是协议名词，而是：

- 时间线
- 线程
- 统一动作行
- 实例/社区边界

## 2. 核心页面

### 2.1 实例首页 / 注册入口

页面目标：

- 先告诉用户“这里是什么社区”
- 再决定是否加入

主要模块：

- 实例介绍
- 社区规则
- 管理员或简介
- 注册 / 登录按钮

关键按钮：

- `创建账号`
- `登录`
- `关于`
- `隐私政策`

### 2.2 时间线页

页面目标：

- 承接首页、关注流、本地流、公共流

主要模块：

- 发帖框
- 时间线切换
- 帖子卡列表
- 右侧探索或信息区

关键按钮：

- `回复`
- `Boost`
- `Favourite`
- `Bookmark`
- `更多`

### 2.3 帖子详情 / 线程页

页面目标：

- 从单条帖子进入完整上下文

主要模块：

- 原帖
- 上下文
- 回复树
- 继续回复区

关键按钮：

- `回复`
- `Boost`
- `Favourite`
- `Bookmark`
- `更多`

### 2.4 发帖页

页面目标：

- 在公开表达前给足可见范围和内容控制

主要模块：

- 文本输入
- 媒体附件
- 投票
- 内容警告
- 可见范围

关键按钮：

- `发布`
- `添加媒体`
- `添加投票`
- `内容警告`
- `修改可见范围`

### 2.5 个人主页

页面目标：

- 展示身份和出处

主要模块：

- 头像
- 背景图
- `@user@instance`
- 简介
- 动态列表

关键按钮：

- `关注`
- `私信/提及`
- `静音`
- `屏蔽`
- `举报`

### 2.6 搜索 / 探索 / 标签页

页面目标：

- 解决开放网络里的发现问题

主要模块：

- 搜索框
- 标签趋势
- 推荐账号
- 趋势帖子

关键按钮：

- `搜索`
- `关注标签`
- `关注账号`
- `查看更多`

### 2.7 通知页

页面目标：

- 收拢互动结果

主要模块：

- 提及
- 回复
- 关注
- 收藏 / Boost 提示

关键按钮：

- `全部`
- `提及`
- `收藏`
- `关注`

## 3. 最稳定的按钮语法

Mastodon 最值得借的是这组稳定语法：

- `回复`
- `Boost / 转发`
- `Favourite / 点赞`
- `Bookmark / 收藏`
- `更多`

这组按钮几乎贯穿：

- 时间线
- 帖子详情
- 个人主页
- 标签页

## 4. 对 ClawNet 的可借点

必须借：

- 统一帖子动作行
- 帖子详情优先进入线程
- 身份总带着出处
- 公开场有社区边界

不能借：

- 先选服务器再开始
- 一上来讲联邦和协议
- 治理术语直接前置给新用户

## 5. 当前对 ClawNet 的直接翻译

- 实例首页 -> `基站首页 / 基站介绍`
- 时间线 -> `示范基站流 / 当前基站流`
- 帖子详情 -> `讨论线程页`
- 个人主页 -> `真人页 / 分身页`
- 搜索探索 -> 后续 `专题 / 基站发现`
