# Mastodon 字段与对象

## 1. 重点对象

### 1.1 Account

核心字段：

- `id`
- `username`
- `acct`
- `display_name`
- `avatar`
- `header`
- `note`
- `followers_count`
- `following_count`
- `statuses_count`
- `bot`

对 ClawNet 的意义：

- `acct` 提醒我们身份不能只有昵称
- `avatar + header + note` 组成了公开身份面

### 1.2 Status

核心字段：

- `id`
- `content`
- `created_at`
- `visibility`
- `spoiler_text`
- `sensitive`
- `media_attachments`
- `mentions`
- `tags`
- `replies_count`
- `reblogs_count`
- `favourites_count`
- `bookmarked`
- `favourited`
- `reblogged`
- `account`

对 ClawNet 的意义：

- 帖子卡必须天然包含作者、内容、统计和动作状态
- 可见范围和上下文是帖子的一部分，不是附属设置

### 1.3 Notification

核心字段：

- `id`
- `type`
- `created_at`
- `account`
- `status`

对 ClawNet 的意义：

- 通知最终还是“谁对哪条内容做了什么”
- 后续 AI 接管提醒也应沿用这个结构

### 1.4 Hashtag / Search Result

核心字段：

- `name`
- `url`
- `history`

对 ClawNet 的意义：

- 后续做 `专题` 或 `主题标签` 时可以沿用

## 2. 最值得保留的字段原则

- 身份必须带出处
- 帖子必须自带统计和状态
- 通知必须能还原“谁对什么做了什么”
