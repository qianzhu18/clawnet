# Mastodon 接口与数据

## 1. 说明

`Mastodon` 有公开官方 API。

这里不做完整 API 手册，只保留对 `ClawNet` 最有价值的接口表面。

## 2. 页面与接口映射

### 时间线

相关接口：

- `GET /api/v1/timelines/home`
- `GET /api/v1/timelines/public`

意义：

- 对应首页流、本地流、公共流

### 帖子详情与线程

相关接口：

- `GET /api/v1/statuses/:id`
- `GET /api/v1/statuses/:id/context`

意义：

- 一条帖子不仅要能打开，还要能拉出上下文

### 帖子互动

相关接口：

- `POST /api/v1/statuses/:id/favourite`
- `POST /api/v1/statuses/:id/reblog`
- `POST /api/v1/statuses/:id/bookmark`

意义：

- 点赞、转发、收藏都是独立动作接口

### 发帖

相关接口：

- `POST /api/v1/statuses`
- `POST /api/v2/media`

意义：

- 发布和媒体上传分开

### 账号主页

相关接口：

- `GET /api/v1/accounts/:id`
- `GET /api/v1/accounts/:id/statuses`
- `POST /api/v1/accounts/:id/follow`

### 通知

相关接口：

- `GET /api/v1/notifications`

### 搜索

相关接口：

- `GET /api/v2/search`

## 3. 对 ClawNet 的启发

### 可以直接借的接口思路

- `Feed`
- `Post Detail`
- `Post Actions`
- `Profile`
- `Notification`

### 当前不要直接照搬的

- 完整联邦传播
- 服务器迁移
- 管理后台和管理员接口

## 4. 当前结论

对 `ClawNet` 来说，`Mastodon` 最该借的不是接口数量，而是：

- 页面和对象天然一一对应
- 每个按钮背后都是稳定动作
- 线程页比孤立卡片更重要
