# 设计交付包

这里不是仓库内部工作台。

这里是后续可以单独发给外部设计师的文档包入口。

## 这一包文档的目标

解决一个问题：

- 当你只上传几份 Markdown 给设计师，而不是把整个仓库都给他时，他还能不能独立理解并继续出图

这套文档必须满足：

- 自洽
- 独立
- 不依赖仓库其他文件
- 不要求设计师先看代码
- 不要求设计师再翻别的文档补上下文

## 当前阅读顺序

1. `界面设计交付规则.md`
2. `页面设计稿模板.md`
3. `交付前检查清单.md`

## 当前使用方式

后续如果你要把页面设计需求发给外部设计师：

1. 先用 `页面设计稿模板.md` 写单页稿
2. 再按 `交付前检查清单.md` 自查
3. 只把这一包文档发出去

## 当前边界

这一包文档不负责：

- 仓库协作规则
- 开发规范
- 历史讨论过程
- 未公开的内部资料引用

它只负责让对方看懂并继续出图。
