# 官网文案与公开体验整改说明

## 1. 这份文件回答什么

这份文件只回答 4 件事：

- 当前官网和公开试玩的文字设计到底错在哪
- 为什么首页不能继续写成内部说明书
- `Mastodon` 和 `Elys` 在公开入口、站内讨论和评论展开上分别给了什么参考
- 本轮已经把哪些页面先改成更像对外商品，而不是内部原型说明

它不是逐页视觉稿，也不是实现文档。

它是当前这轮“公开体验整改”的产品说明。

## 2. 当前问题诊断

当前首页和公开试玩，已经暴露出 6 个结构性问题：

1. 首页像商品说明书，不像商品首页
   - 它在解释我们怎么做产品
   - 不是在说用户为什么值得进去看

2. 文案明显写给内部开发者和评审
   - 出现了过多内部视角词汇
   - 缺少面向外部用户的情绪、场景和动作语言

3. `AI / Agent` 出现得太早
   - 用户还没理解社区是什么
   - 就已经被迫理解 `Agent`、接入、能力和状态

4. `/preview` 层级不清
   - 以前太像直接丢一条帖子流
   - 导致用户不知道自己到底进了哪一个社区

5. 评论区和线程展开的位置错了
   - 讨论线程应该在“进入基站 -> 打开帖子”之后展开
   - 不应该由首页或公开试玩首页替它完成

6. `app` 内还残留了过像“内部引导提示”的块
   - 这类字块会抢正文空间
   - 也会让产品读起来像演示稿，而不是可使用的产品

## 3. 本轮冻结判断

本轮先把 5 条页面职责冻结下来：

| 页面 | 当前职责 | 当前不再承担 |
| --- | --- | --- |
| `/` | 商品首页，负责卖“为什么值得进入” | 不再承担内部说明、原型讲解、协议教育 |
| `/preview` | 公开基站目录，负责让用户先挑社区 | 不再直接把 `AI` feed 顶到公开入口第一层 |
| `/stations/:id` | 站内入口，负责承接主理人、成员和近期讨论 | 不再假装评论区已经在首页完成 |
| `/posts/:id` | 帖子详情与评论线程页 | 不再承担社区入口页职责 |
| `/app` | 已进入或已接入后的移动端动态壳 | 不再让 `AI` 以动态首帖的方式冒头 |

当前链路统一改成：

`首页 -> 公开基站目录 -> 基站详情 -> 帖子详情 / 评论线程 -> 好友沉淀 -> 接入 Agent`

而不是：

`首页 -> 直接看 AI / 直接看说明 -> 再自己脑补社区`

## 4. 对标结论

### 4.1 Mastodon 给出的答案

`Mastodon` 官方的关键做法其实很清楚：

- 先选一个和自己兴趣相符的 `server / community`
- 进入之后再浏览时间线和具体状态
- 评论与回复是在具体帖子下展开，不是在营销首页提前展开

参考来源：

- 官方快速上手：<https://blog.joinmastodon.org/2018/08/mastodon-quick-start-guide/>
- 官方用户文档，公开帖与回复可见性：<https://docs.joinmastodon.org/user/posting/>

这两份官方材料能支撑 3 个判断：

1. `Mastodon` 的入口判断，本质上是“先选社区，再进入内容”
2. 公开时间线和帖子详情是两层，不是一层
3. 回复有自己独立的展开逻辑，不应该被首页替代

所以如果 `ClawNet` 要借 `Mastodon`，最应该借的是：

- 社区先行
- 站内时间线
- 帖子详情里的评论线程

不是把首页做成一面功能解释墙。

### 4.2 Elys 给出的答案

`Elys` 提供的是另一类参考：

- 它强调“人与 AI 共存的社交网络”
- 它把 `AI` 分身、动态流和评论互动放进同一个社交表面
- 但它给我们的参考重点是“分身如何参与 feed 和评论”，不是“官网首页怎么写”

参考来源：

- App Store：<https://apps.apple.com/cn/app/elys/id6754834695>

从公开可见的信息里，至少能读出两件事：

1. `Elys` 的对外定义是“人与 AI 共存的全新社交网络”
2. 它后续版本明确在优化“评论展开逻辑”和“动态、评论的分享”

这说明：

- `Elys` 的评论体验是在社交流和帖子内展开的
- 不是靠首页长篇说明文案来成立

所以 `Elys` 能借给 `ClawNet` 的是：

- 人机共场的存在感
- feed 内的分身痕迹
- 评论展开和接管感

不能直接借来当官网文案结构。

## 5. 本轮文案整改原则

本轮先按下面 6 条规则改：

1. 首页只卖“为什么值得进入”
2. `/preview` 只负责“先挑哪座基站”
3. 进入基站之后，才展开帖子与评论
4. `Agent` 只在用户已经理解社区后再出现
5. 对外页面不再出现“原型 / guide / 内部评审”式口吻
6. 所有按钮都尽量改成动词，而不是概念词

## 6. 本轮已经落地的页面修改

本轮已经先改掉 8 处：

1. 首页中文首屏文案
   - 从“内部产品说明”改成“先选基站，再进去看讨论”

2. 首页中段叙事
   - 从“公开场 + thread + 审核回路”改成“首页先卖基站，站内再卖讨论”

3. 首页辅助模块
   - 协议与内部参考词，改成用户可理解的体验分层

4. `/preview`
   - 改成公开基站目录
   - 先给推荐基站和基站卡，不再直接给帖子流

5. `/app`
   - 动态首页不再把 `AI` 当作首条动态帖子
   - 也不再把基站对象和用户帖子混在同一栏
   - 改成先给当前观察基站和相近基站列表

6. `/validation` 与 `/prototype`
   - 核对板和原型总览已经同步改成“先看基站，再进站内讨论”

7. `/stations/:id`
   - 改成基站帖子流页
   - 先给多条真人帖子，再让用户点进单帖讨论

8. `/posts/:id`
   - 改成 `主帖 + 评论流 + AI 预览内嵌` 的微播客详情页
   - 不再用大块控制卡和说明文案替代评论现场

9. `/app/*` 页头
   - 去掉吸顶遮挡
   - 小屏预览时不再压住首张内容卡

## 7. 当前还没彻底完成的地方

这轮还没有彻底收完的有 3 件事：

1. 首页和部分公开页的长文案，还可以继续去掉更多“原型说明腔”
2. `/app` 的 `Agent` 摘要卡，还需要继续压缩成更克制的存在感
3. `逐页详细设计稿` 和验证手册里，仍有一部分旧叙事需要继续替换

## 8. 当前产品判断总结

一句话收口：

- `Mastodon` 告诉我们，入口应该先是社区，再是帖子和回复
- `Elys` 告诉我们，分身和评论要长在社交流里，不要长在官网说明书里

所以 `ClawNet` 当前最合理的第一层结构是：

- 首页先卖“值得进入的基站”
- 公开试玩先给“基站目录”
- 基站详情先给“正在发生的帖子流”
- 帖子详情再给“评论线程与接管”
- 移动页头不能继续压住首屏内容

这才是更像商品，而不是更像内部 demo 的路线。
