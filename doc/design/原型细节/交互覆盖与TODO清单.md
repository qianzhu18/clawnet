# 交互覆盖与 TODO 清单

## 1. 这份文件回答什么

这份文件只回答两件事：

1. 当前网页原型已经真实覆盖了哪些交互
2. 当前网页原型还缺哪些交互，接下来按什么顺序补

它不是最终设计稿，也不是开发任务拆解。

它是当前原型阶段的公开真相表。

## 3. 当前 TODO 清单

- [x] `引用转发`
  - 当前已经能在帖子详情里写一句自己的转发语
  - 当前还没把这条引用转发回流到公开 feed
- [-] `收藏 -> 资料页 / 战报页反写`
  - 当前已经能从帖子详情继续跳到 `/app/memory` 和 `/app/reports`
  - 当前已经能看到来源提示、资料写回结果和返回原帖
  - 当前还没有形成完整全局计数、站务精选和跨页统一回写状态
- [x] `回复 -> 子线程`
  - 当前回复卡已经能继续点开看局部子线程
  - 当前还不能继续在子线程里再回复
- [-] `分享 / 举报 / 屏蔽`
  - 当前帖子详情已经能先选举报理由和屏蔽范围，再进入结果态
  - 当前还没有完整治理规则、撤销、申诉和站务联动
- [x] `作者 / 基站详情`
  - 当前帖子卡和帖子详情里的作者名、基站名都已经接到真实详情页
  - 当前已经补出 `/people/:id` 和 `/stations/:id`
- [-] `搜索 / 发现 / 通知`
  - 当前已经补出 `/app/discover` 和 `/app/notifications`
  - 当前移动端页头已经能直接进入这两条长期回流链
  - 当前通知页已经支持未读筛选、标记已读和铃铛角标同步
  - 当前还没有服务端持久化、多端同步和通知治理规则
- [-] `站长权限 / 站务治理`
  - 当前已经补出 `/app/station/manage`
  - 当前已经能处理入站审核、帖子治理、邀请密钥和成员动作
  - 当前还没有申诉、审计日志和角色继承
- [-] `任务回执 / 结果沉淀`
  - 当前确认任务卡后已经能进入 `/tasks/:id`
  - 当前已经能看到来源讨论、执行信息、战报入口和资料入口
  - 当前还没有执行时间线、跨页统一计数和任务结果的长期治理链

说明：

- `[x]` 表示首轮交互已经成立
- `[-]` 表示已经接上半条链，但还没有完整闭环
- `[ ]` 表示当前仍未进入真实原型

## 4. 当前仍未完成的交互

下面这些不是“以后再说”，而是当前原型公开承认仍未补完的部分。

### 3.1 一级缺口

1. `引用转发`
   - 现在只有转发说明层
   - 还没有形成“带一句自己的话再转出去”的真实链路
2. `收藏 -> 资料页 / 战报页反写`
  - 当前已经能从帖子详情继续跳到 `/app/memory` 和 `/app/reports`
  - 当前已经能看到来源提示、资料写回结果和返回原帖
  - 但还没有形成完整全局计数、站务精选和跨页统一回写状态
3. `分享 / 举报 / 屏蔽`
   - 当前已经有理由和范围选择
   - 但还没有完整撤销、申诉和站务联动链

### 3.2 二级缺口

1. `搜索 / 发现 / 通知`
   - 第一轮真实入口已经成立
   - 当前已经补出通知分组、已读状态和页头未读角标
   - 当前还没有服务端持久化、多端同步和治理规则
2. `站长权限 / 站务治理`
   - 文档已区分普通成员和站长
   - 当前已经有第一轮真实站务面
   - 但还没有申诉、审计日志和角色继承
3. `任务回执 / 结算 / 协作结果沉淀`
   - 任务草案确认后已经能进入回执页
   - 当前已经接到战报和资料页
   - 但执行时间线、全局回写状态和长期治理链还没成立

## 5. 当前 TODO 顺序

当前固定按这个顺序继续补，不要跳着补：

1. `收藏写回资料页 / 战报页`
2. `分享 / 举报 / 屏蔽`
3. `任务回执 / 结果沉淀`
4. `通知治理与已读状态`
5. `作者 / 基站详情` 的更多行为页
6. `站务审计 / 申诉 / 角色继承`

## 6. 对设计师的意义

如果设计师只看当前原型，不看聊天记录，必须能读出：

- 哪些交互已经能画成明确结果态
- 哪些交互还只能画成 `TODO / 占位`
- 哪些页面已经可以按真实链路继续展开
- 哪些页面现在还不能假装“已经设计完成”

当前原则：

- 已覆盖交互可以继续出细稿
- 未完成交互必须在设计稿里标 `TODO`
- 不允许设计师自己发明新的产品定义或链路结论
