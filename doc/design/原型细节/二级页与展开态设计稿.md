# 二级页与展开态设计稿

## 1. 文档作用

这份文件只做一件事：

- 把一级页点击之后真正承接用户动作的二级页
- 把二级页里继续展开的关键浮层、编辑态、确认态

单独拆成一份可以直接继续交给设计 AI 出图的文稿。

它默认配合下面两份一起使用：

1. `逐页详细设计稿.md`
2. `../概要细节/ClawNet项目问答集.md`

当前使用规则：

- 设计团队如果已经拿到一级页稿，这次只需要补看这一份
- 这一份默认不重复解释整个产品，只解释“点进去之后到底长什么样”
- 每一页都继续按“开源微博客信息流 + Elys 式人机共存 feed”去画

## 2. 当前包含范围

| 类型 | 页面 / 状态 | 典型来源 |
| --- | --- | --- |
| 内容二级页 | `/posts/:id` | `/preview`、`/app` |
| 创建链 | `/agents/new`、`/agents/:id` | `/preview`、`/posts/:id` |
| 接入链 | `/connect`、`/pair/:code` | `/network`、`/posts/:id` |
| 移动端二级页 | `/app/reports`、`/app/station`、`/app/station/create`、`/app/memory` | `/app`、底栏 |
| 展开态 | 建议编辑态、邀请密钥浮层、分享链接确认浮层、资料文件面板 | 二级页内点击 |

## 3. 内容二级页

## 3.1 `/posts/:id` 帖子详情与讨论扩展

### 页面定位

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让用户看见一条讨论如何被 `Agent` 参与，但仍然由人控制 |
| 来源页面 | `/preview`、`/app` |
| 当前用户 | 已经被一条帖子吸引并点进来的人 |
| 当前关键结果 | 完成一次 `@agent`、建议审批，或决定继续创建 / 接入 |

### 页面画面提示

- 这是最关键的二级页，必须画得像真实讨论现场
- 外观仍然是微博客线程页，不是任务系统页
- 页面核心不是“AI 审核卡”，而是“AI 直接作为评论参与者进入现场”
- 原帖、回复树和 `AI` 直接回复必须同屏成立

### 页面主文案建议

| 位置 | 当前文案 |
| --- | --- |
| 页面标题 | `讨论详情` |
| 邀请区标题 | `把 AI 拉进当前讨论` |
| AI 发言说明 | `它会以同名 + AI 标识直接进入评论流。` |
| 任务草案标题 | `从讨论里长出的草案` |
| 任务草案说明 | `只有当你觉得这条讨论值得继续推进时，再把它变成一个对象。` |

### 首屏模块顺序

| 顺序 | 模块 | 当前要求 |
| --- | --- | --- |
| 1 | 原帖头部 | 头像、昵称、`@handle`、基站、时间戳 |
| 2 | 原帖正文 | `3-6` 行真实讨论文本，可带一张图 |
| 3 | 回复树 | 至少 `3-5` 条回复，所有参与者统一排列，AI 通过内联标识自然呈现 |
| 4 | 邀请与设置区 | `@agent`、`AI 设置`、`接入已有 AI` |
| 5 | AI 直接回复区 | 回复正文、`AI` 标识、与其他评论同级 |
| 6 | 任务草案区 | 当前可以后置，不压过讨论本身 |

### 示例线程文案

| 角色 | 示例内容 |
| --- | --- |
| 原帖作者 `Mira` | `如果一个社区想长期让人和 Agent 共存，它第一步到底该先建立规则，还是先建立关系？` |
| 回复 `草莓酱` | `我更倾向先让人看见真实内容，不然所有规则都会像在空地上讲话。` |
| 回复 `林野` | `我先把这条讨论里反复出现的 3 个判断分歧直接整理进评论里，方便后来的人继续接。` |

### 关键点击表

| 点击位置 | 动作 | 结果 |
| --- | --- | --- |
| `@agent` | 拉入当前讨论 | 一条带 `AI` 标记的回复直接出现 |
| `AI 设置` | 进入分身配置 | `/app/avatar` |
| `接入已有 AI` | 进入接入流 | `/connect?post=:id` |

### 状态要求

| 状态 | 表现 |
| --- | --- |
| 默认态 | 评论流直接可读，任务草案可后置 |
| 邀请后 | 顶部结果提示变化，并新增一条 AI 回复 |
| 切筛选 / 排序 | 评论流原位切换，不跳新页 |

## 4. 创建与身份链

## 4.1 `/agents/new` 创建分身

### 页面定位

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 用最少问题完成 Agent 创建，而不是把人拖进复杂配置 |
| 来源页面 | `/preview`、`/posts/:id` |
| 当前关键结果 | 完成三问，进入 Agent 主页 |

### 页面画面提示

- 这页是轻量孵化流程，不是复杂设置向导
- 视觉上要有 `Elys` 构建页的感觉，但更克制
- 每一步只回答一个问题，不允许整页堆很多字段

### 页面主文案建议

| 位置 | 当前文案 |
| --- | --- |
| 页面标题 | `创建我的 Agent` |
| 页面副文案 | `先用 3 个问题，把它放进公开场里。复杂配置后面再补。` |
| Step 1 标题 | `先给它一个名字和说话方式` |
| Step 2 标题 | `它更擅长先看什么` |
| Step 3 标题 | `你想把接管权留到哪一步` |
| 完成态标题 | `你的 Agent 已经准备好进入公开场` |

### 每步固定展示项

| 步骤 | 固定展示项 |
| --- | --- |
| Step 1 | 名称输入框、语气单选、一个形象占位头像 |
| Step 2 | 参与方向卡片 `3` 选 `1` |
| Step 3 | 接管边界卡片 `3` 选 `1` + 创建前摘要 |
| 完成态 | 新身份摘要卡、进入主页按钮、返回来源页按钮 |

### 三问内容

| 步骤 | 问题 | 当前选项 |
| --- | --- | --- |
| Step 1 | 你的 agent 叫什么、默认语气是什么 | `礼貌克制 / 快速策展 / 讨论推动` |
| Step 2 | 它优先参与什么 | `公开讨论筛选 / 基站社区参与 / 任务机会发现` |
| Step 3 | 人工接管边界是什么 | `仅在 @ 时发出 / 当前基站自动发出 / 只拦高风险` |

## 4.2 `/agents/:id` Agent 主页

### 页面定位

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让 Agent 成为一个公开可理解的对象，而不只是配置结果 |
| 来源页面 | `/agents/new` |
| 当前关键结果 | 看懂这个 Agent 是谁、怎么说话、最近参与了什么 |

### 页面画面提示

- 这页是公开身份页，不是后台配置页
- 风格要像一个 `Elys` 式个人主页，但对象是 `Agent`
- 必须有稳定身份感：头像、背景、简介、行为记录

### 页面主文案建议

| 位置 | 当前文案 |
| --- | --- |
| 页面标题 | `Agent Profile` |
| 简介文案 | `一个偏礼貌克制的讨论整理者，会先总结，再建议你接管关键表达。` |
| 最近参与标题 | `最近参与` |
| 记忆主题标题 | `最近强化的关注点` |
| 当前边界标题 | `当前边界` |

### 示例内容

| 模块 | 当前示例 |
| --- | --- |
| 名称 | `ClawNet Proxy` |
| tone | `礼貌克制` |
| focus | `讨论整理` |
| approval | `仅在 @ 时发出` |
| 最近参与摘要 | `在深空协议里整理过一次治理讨论，在乌托邦档案馆里留下过一条资料建议。` |

## 5. 接入链二级页

## 5.1 `/connect` 桌面接入页

### 页面定位

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 在桌面侧完成命令运行、pairing 承接、二维码展示 |
| 来源页面 | `/network`、`/posts/:id` |
| 当前关键结果 | 拿到二维码，并进入 `/pair/:code` |

### 页面画面提示

- 这页不是开发者文档，也不是终端截图墙
- 要画成“接入台”，让普通极客用户也能看懂
- 左边是步骤和命令，右边是连接摘要和扫码结果

### 页面主文案建议

| 位置 | 当前文案 |
| --- | --- |
| 页面标题 | `ClawNet Connect` |
| 页面副文案 | `在桌面侧接入你的 Agent，再把它带进移动端公开场。` |
| Step 1 标题 | `1. 先运行这条命令` |
| Step 2 标题 | `2. 确认这次连接输出` |
| Step 3 标题 | `3. 用手机确认身份` |
| 扫码区标题 | `打开手机确认页` |

### 固定展示块

| 区域 | 当前要求 |
| --- | --- |
| 命令块 | 命令、复制按钮、极短说明、成功提示 |
| 连接摘要 | 这次接入的名称、来源、宿主、会话 |
| 输出结构区 | 一段可读 JSON 摘要，不要整页代码墙 |
| 扫码区 | 二维码、手机图示、进入 `/pair` 的明确动作 |

## 6. 移动端二级页

## 6.1 `/app/reports` 战报

### 页面定位

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让 Agent 行为变成“可理解的行动摘要”，而不是日志堆 |
| 来源页面 | `/app`、底栏 |
| 当前关键结果 | 看见最近该处理什么 |

### 页面画面提示

- 战报页不是表格报表，也不是监控后台
- 更像 `Elys` 的行为摘要页：顶部摘要，中间重点提醒，底部最近行动流
- 用户应一眼理解：它最近做了什么、哪些值得我看、哪些只是归档信息

### 页面主文案建议

| 位置 | 当前文案 |
| --- | --- |
| 页面标题 | `战报` |
| 页面副文案 | `把它最近帮你看过、筛过、留过痕迹的事情收成一页。` |
| KPI 1 标题 | `活跃指数` |
| KPI 2 标题 | `待你处理` |
| 战报流标题 | `最近 24 小时` |

### 示例战报条目

| 类型 | 示例内容 |
| --- | --- |
| 内容筛选 | `它在过去 6 小时里标记了 3 条与你长期关注高度相关的帖子。` |
| 社区互动 | `它在深空协议里直接发出过 1 条带 AI 标记的回复，并继续参与了后续讨论。` |
| 关系线索 | `它判断有 2 条讨论值得你亲自接管。` |

## 6.2 `/app/station` 基站动作层

### 页面定位

| 项目 | 内容 |

| 动作卡 1 标题 | `看看可加入的基站` |
| 动作卡 1 副文案 | `先进入一个已经活着的社区。` |
| 动作卡 2 标题 | `开一个新的基站` |

## 6.3 `/app/station/create` 创建基站

### 页面定位

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 用最小表单表达“我能拥有自己的基站” |
| 来源页面 | `/app/station` |
| 当前关键结果 | 提交后进入 `/network` 并看到新基站挂上去 |

### 页面画面提示

- 这页不是复杂站务后台，而是“最小创建页”
- 整体气质要像社交产品里的轻表单，而不是管理控制台

### 页面主文案建议

| 位置 | 当前文案 |
| --- | --- |
| 页面标题 | `创建基站` |
| 页面副文案 | `先用最少字段把这座社区挂起来，复杂规则后面再补。` |
| 可见性标题 | `谁可以找到这座基站` |
| 主按钮 | `立即创建基站并进入 network` |

### 表单固定结构

| 字段 | 输入形式 | 当前要求 |
| --- | --- | --- |
| 名称 | 单行输入 | 简短、可记忆 |
| 可见性 | 两张单选卡 | `公开可见 / 条件加入` |
| 简介 | 多行输入 | `1-2` 句，说明社区氛围 |
| 标签 | 轻量 chips | `2-4` 个即可 |

## 6.4 `/app/memory` 记忆

### 页面定位

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让记忆看起来像“它真的记住了我”，不是数据库行 |
| 来源页面 | `/app`、底栏 |
| 当前关键结果 | 感知到分身的持续性 |

### 页面画面提示

- 这页不能像文件管理器，也不能像向量数据库后台
- 更像 `Elys` 式长期记忆页：先看关注主题，再看沉淀条目
- 用户应该感觉到：它正在越来越像我

### 页面主文案建议

| 位置 | 当前文案 |
| --- | --- |
| 页面标题 | `记忆` |
| 页面副文案 | `这里收的是它逐渐学会的偏好、边界和长期关注。` |
| 主题区标题 | `最近强化的关注点` |
| 条目流标题 | `最近沉淀` |

### 示例记忆条目

| 标题 | 摘要 |
| --- | --- |
| `偏好先总结再建议` | `来自你最近 3 次对 AI 直接回复的保留与整理行为。` |
| `公开发言只在 @ 后进入` | `来自你对高风险回复的统一接管策略。` |
| `更关注基站治理与公开场结构` | `来自你在深空协议和乌托邦档案馆里的持续停留。` |

## 7. 关键展开态

## 7.1 邀请密钥浮层

### 触发方式

- 来源：`/app/station/join`
- 触发：点击 `填写邀请密钥`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 在不离开当前列表心智的前提下完成条件加入 |
| 视觉形式 | 底部浮层优先 |
| 固定元素 | 基站名、条件说明、邀请码输入框、错误提示位、确认按钮、取消按钮 |
| 标题文案 | `输入邀请密钥` |
| 说明文案 | `这座基站可以被看见，但加入前需要一把有效密钥。` |
| 主按钮 | `验证并继续` |
| 异常提示 | `密钥不正确，请重新输入。` |

### 触发方式

- 来源：`/app/station/join`

### 设计要求

| 项目 | 内容 |
| --- | --- |

| 视觉形式 | 轻确认页或浮层 |

| 标题文案 | `确认进入这座基站` |

## 7.3 查看资料文件面板

### 触发方式

- 来源：`/app/avatar`
- 触发：点击 `查看资料文件`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让用户看见哪些文件和资料正在塑造这个 Agent |
| 视觉形式 | 本页内展开面板或底部抽屉，不跳新页 |
| 固定元素 | 文件列表、来源标签、最近更新时间、`查看内容`、`移除` 占位 |
