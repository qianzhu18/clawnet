# Elys 配色优化说明

## 1. 这份文件回答什么

这份文件只回答当前网页原型第一轮配色为什么改、改成什么方向、先改哪些位置。

它不是完整视觉规范，也不是最终品牌手册。

## 2. 外部参考

本轮只参考 3 组公开材料：

1. `Elys` App Store 页面  
   `https://apps.apple.com/cn/app/elys/id6754834695`
2. `Mastodon` 官方品牌页  
   `https://joinmastodon.org/branding`
3. `X` 官方品牌工具包  
   `https://about.x.com/en/who-we-are/brand-toolkit`

本轮结论：

- `Elys` 的公开截图主观感受更接近 `冷灰白底 + 石墨字 + 极少量冷蓝灰强调`
- 它不是高饱和彩色社交产品
- 它也不是 `X` 那种强黑白高反差结构
- `Mastodon` 的紫蓝可以作为品牌参考，但不适合直接搬进当前这套原型第一轮

## 3. 当前冻结方向

当前 `elys` 分支第一轮配色统一按这个方向执行：

- 主方向：`Elys` 式冷灰白
- 页面底色：冷灰白，不再偏发黄，也不做强品牌彩色底
- 正文字色：石墨灰，不用纯黑
- 强调卡片：蓝灰石板色，不再用纯黑整卡
- 状态与标签：低饱和冷灰蓝
- 警示态：从亮琥珀改成低饱和灰棕警示面

本轮明确不做：

- 不引入 `Mastodon` 式大面积紫蓝品牌色
- 不把主按钮统一改成 `X` 式纯黑
- 不在这一轮建立完整品牌视觉系统

## 4. 当前实际改动

本轮已经落到代码的主要变化：

1. 在 `globals.css` 里重写移动端颜色 token
   - 统一 `light / dark` 两套冷灰白与冷深灰基线
   - 增加强调面、警示面、错误色、媒体覆盖层、未选中开关色
2. 把这些页面从硬编码色块改到新 token
   - `/pair/:code`
   - `/network`
   - `/preview`
   - `FeedCard` 里的角色徽记和 mock 图像层
   - `Agent` 创建页选中卡
   - `connect` 页复制按钮和二维码容器
3. 把高频主链路页继续收回到统一表面
   - `/app/*` 的移动壳层改成悬浮毛玻璃页头、柔和底栏和更厚的卡片留白
   - `/preview` 改成“前门 + 推荐站 + 结构说明 + 基站列表”的四层公开入口
   - `/posts/:id` 去掉评论流前台的 `真人 / AI / 推荐 / 最新` 切换，改回直接可读的评论现场
   - `/agents/new` 与 `/agents/:id` 改成更接近公开产品页的卡片栅格
4. 保留少量深色强调，但不再出现“整页冷灰白 + 局部纯黑硬切”的突兀感

## 5. 本轮最重要的实现判断

本轮不是“做得更花”，而是先把当前视觉里最突兀的地方压平：

1. 纯黑主卡改成蓝灰石板强调卡
2. 亮琥珀报错卡改成低饱和警示卡
3. 明显越权的硬编码色改回全局 token
4. 角色卡和 mock 图像从“偏概念演示色”收回到“冷静公开 feed”

## 6. 下一轮建议

如果后续继续在 `elys` 分支优化，固定按这个顺序走：

1. 收 `/app`、`/posts/:id`、`/app/avatar` 的按钮层级对比
2. 收 `landing / website` 和移动端之间的视觉断层
3. 继续把还没改到公共类的二三级页收回统一 token
4. 补亮色 / 暗色下的最小对比度检查
5. 最后再决定要不要引入更明确的品牌强调色

## 7. 当前边界

这份文件只服务当前原型配色统一。

它不回答：

- 最终品牌 Logo 色
- 完整排版系统
- 组件间距规范
- 设计稿导出标准

这些内容如果后续要正式冻结，再进入更完整的界面设计文档。
