# Elys 配色优化说明

## 1. 这份文件回答什么

这份文件只回答当前网页原型第一轮配色为什么改、改成什么方向、先改哪些位置，以及当前多色系如何落到代码里。

它不是完整视觉规范，也不是最终品牌手册。

## 2. 外部参考

本轮只参考 3 组公开材料：

1. `Elys` App Store 页面  
   `https://apps.apple.com/cn/app/elys/id6754834695`
2. `Mastodon` 官方品牌页  
   `https://joinmastodon.org/branding`
3. `X` 官方品牌工具包  
   `https://about.x.com/en/who-we-are/brand-toolkit`

本轮结论：

- `Elys` 的公开截图主观感受更接近 `冷灰白底 + 石墨字 + 极少量冷蓝灰强调`
- 它不是高饱和彩色社交产品
- 它也不是 `X` 那种强黑白高反差结构
- `Mastodon` 的紫蓝可以作为品牌参考，但不适合直接搬进当前这套原型第一轮

## 3. 当前冻结方向

当前 `elys` 分支第一轮配色统一按这个方向执行：

- 主方向：`Elys` 式冷灰白
- 页面底色：冷灰白，不再偏发黄，也不做强品牌彩色底
- 正文字色：石墨灰，不用纯黑
- 强调卡片：蓝灰石板色，不再用纯黑整卡
- 状态与标签：低饱和冷灰蓝
- 警示态：从亮琥珀改成低饱和灰棕警示面
- 多色系也只允许在这个基线内变化：
  - `default`：霜蓝
  - `sakura`：柔粉
  - `mint`：清绿
  - `lavender`：浅紫
- 四套主题都必须保持低饱和、轻发光、弱对比，不做高饱和彩壳

本轮明确不做：

- 不引入 `Mastodon` 式大面积紫蓝品牌色
- 不把主按钮统一改成 `X` 式纯黑
- 不在这一轮建立完整品牌视觉系统

## 4. 当前实际改动

本轮已经落到代码的主要变化：

1. 在 `globals.css` 里重写移动端颜色 token
   - 统一 `light / dark` 两套冷灰白与冷深灰基线
   - 增加强调面、警示面、错误色、媒体覆盖层、未选中开关色
2. 把这些页面从硬编码色块改到新 token
   - `/pair/:code`
   - `/network`
   - `/preview`
   - `FeedCard` 里的角色徽记和 mock 图像层
   - `Agent` 创建页选中卡
   - `connect` 页复制按钮和二维码容器
3. 把高频主链路页继续收回到统一表面
   - `/app/*` 的移动壳层改成悬浮毛玻璃页头、柔和底栏和更厚的卡片留白
   - `/preview` 改成“前门 + 推荐站 + 结构说明 + 基站列表”的四层公开入口
   - `/posts/:id` 去掉评论流前台的 `真人 / AI / 推荐 / 最新` 切换，改回直接可读的评论现场
   - `/agents/new` 与 `/agents/:id` 改成更接近公开产品页的卡片栅格
4. 接入多色系底座
   - 通过 `next-themes` 把主题名写到 `html[data-theme]`
   - 在 `globals.css` 中用 `HSL` 变量定义 `default / sakura / mint / lavender`
   - 用 Tailwind v4 的 `@theme inline` 暴露 `theme-primary / theme-hover / theme-light / shadow-theme-glow`
   - 把公共按钮、标签、强调卡、移动端发光和全局背景统一改成从主题变量取色
5. 把微播客高对比规则继续写进当前信息流
   - 新增 `app / content / action` 三组 token，固定正文、辅助文字、动作色和信息流底色
   - feed 卡片从厚边框和整卡阴影收回到 `白底 + 极细底边线 + hover 轻底色`
   - 评论 / 转发 / 点赞动作分别改成蓝 / 绿 / 红 hover 反馈
6. 保留少量深色强调，但不再出现“整页冷灰白 + 局部纯黑硬切”的突兀感

## 5. 本轮最重要的实现判断

本轮不是“做得更花”，而是先把当前视觉里最突兀的地方压平：

1. 纯黑主卡改成蓝灰石板强调卡
2. 亮琥珀报错卡改成低饱和警示卡
3. 明显越权的硬编码色改回全局 token
4. 角色卡和 mock 图像从“偏概念演示色”收回到“冷静公开 feed”
5. 多主题只换同一套结构里的主色、浅色和发光，不额外改布局和组件语法
6. 信息流里的功能色不再完全跟主题色走，而是按微播客习惯保留蓝 / 绿 / 红三种功能反馈

## 6. 当前实现方法

当前代码固定按这条链路落：

1. `globals.css`
   - 定义四套 `HSL` 主题变量
   - 因为当前项目是 Tailwind v4，所以不走 `tailwind.config.ts`，而是用 `@theme inline` 直接暴露颜色和阴影 utility
   - 把主题色继续映射到公共背景 glow、移动端按钮、标签和强调卡
   - 另外单独定义 `app / content / action`，负责高对比正文和功能动作色
2. `next-themes`
   - 只负责把主题名稳定写到 `data-theme`
   - 不让业务组件各自存一份主题状态
3. `ThemeSwitcher`
   - 只负责切换 `data-theme`
   - 不负责改页面结构
4. `mobile-shell`
   - 继续保留当前 `明亮 / 暗色` 切换
   - 同时把亮暗状态同步到根节点 `.dark`，让多色系在暗态下也能提亮主色
5. `FeedCard / PostDetail / ReplyCard`
   - 改回更接近微博客的扁平信息流
   - 不再默认依赖厚阴影卡片做层级
   - 动作条按评论蓝、转发绿、点赞红来做 hover

## 7. 下一轮建议

如果后续继续在 `elys` 分支优化，固定按这个顺序走：

1. 收 `/posts/:id` 的最后 20% 微播客质感
   - 评论头像列的极细连接线
   - 底部输入框的悬浮胶囊化
   - Bottom Sheet 的拖拽条、柔化遮罩和大圆角收口
   - 标题 / 正文 / 辅助信息 / 未激活动作的固定色阶纪律
2. 收 `/app`、`/app/avatar` 的按钮层级对比
3. 收 `landing / website` 和移动端之间的视觉断层
4. 继续把还没改到公共类的二三级页收回统一 token
5. 补亮色 / 暗色下的最小对比度检查
6. 最后再决定要不要引入更明确的品牌强调色

## 8. 当前新增的视觉纪律

为了把当前帖子详情页继续拉近到顶级 `Elys` 微播客质感，这一轮再补 4 条硬规则：

1. `Thread` 线索必须看得见
   - 主贴头像列和需要承接上下文的回复头像列，要补极细连接线
   - 连接线只能是极弱引导，不能长成论坛式粗树
2. 底部输入框必须像悬浮胶囊
   - 允许磨砂、弱描边、极轻阴影
   - 不允许继续用厚重整块 panel 和大体积投影压底
3. Bottom Sheet 必须像系统浮层
   - 遮罩更柔和
   - 面板上缘更圆
   - 顶部优先拖拽条和居中标题，不再让“关”按钮抢第一眼
4. 文字色阶必须严格稳定
   - 昵称 / 大标题：最深一级
   - 正文：次深一级，但继续高可读
   - `@handle` / 时间戳 / 指标数字：再降一级
   - 未激活动作图标：最浅一级
   - 页面非按钮区域避免大面积实色底板，只保留极低透明度弱底色

## 9. 当前边界

这份文件只服务当前原型配色统一。

它不回答：

- 最终品牌 Logo 色
- 完整排版系统
- 组件间距规范
- 设计稿导出标准

这些内容如果后续要正式冻结，再进入更完整的界面设计文档。
