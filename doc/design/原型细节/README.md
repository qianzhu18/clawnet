## 当前边界

这里仍然只负责：

- 原型结构
- 页面模块
- 按钮与跳转
- mock 数据
- 字段级文案

- 字体系统

## 各文件职责

### `原型审阅指南.md`

告诉你：

- 审稿顺序
- 每页重点看什么

### `逐页详细设计稿.md`

告诉你：

- 每页有哪些模块
- mock 数据怎么放
- 按钮点完去哪里

它不是最终的界面设计稿。

### `二级页与展开态设计稿.md`

告诉你：

- 一级页点进去之后的二级页怎么画
- 关键浮层、编辑态、确认态怎么画
- 设计团队第二轮补稿时先看什么

### `三级展开态设计稿.md`

告诉你：

- 二级页里面继续点开的结果怎么画
- 深层编辑态、异常态、成功态怎么画
- 设计团队补三级交互时先看什么

### `四级字段与反馈态设计稿.md`

告诉你：

- 字段名、placeholder、helper 文案怎么写
- 空态、无结果态、错误态、成功反馈怎么写
- 同页点击之后的微状态差异怎么画

### `交互覆盖与TODO清单.md`

告诉你：

- 当前哪些交互已经真实成立
- 当前哪些交互仍然只是 `TODO`
- 继续补交互时应按什么顺序推进

### `本机可视化交互验证手册.md`

告诉你：

- `3000` 端口怎么验证
- `/prototype` 和 `/validation` 怎么用

### `页面原型拆解模板.md`

这是模板，不是当前主阅读稿。

只有当你要继续新拆某一页的原型结构时，再打开。
