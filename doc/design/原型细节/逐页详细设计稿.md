# 逐页详细设计稿

## 1. 文档定位

这份文件不是再讲一遍“产品方向”。

阅读顺序固定为：

1. `../标准文档/1-需求分析.md`
2. `../标准文档/2-概要设计.md`
3. `../标准文档/3-产品原型设计.md`
4. `原型审阅指南.md`
5. 最后再看这份

它只做 4 件事：

- 把当前真实原型的每一个页面拆到可继续出 UI 稿的粒度
- 明确每页首屏先出现什么、模块怎么排
- 明确每个按钮点完去哪里、带什么状态
- 明确当前用什么 mock 数据表达页面成立

使用规则：

- 这份文档只基于当前代码里真实存在的页面写
- 不把候选壳层写成现状
- 后续设计稿和前端细化，默认都从这份文档继续往下走

## 3. 全局设计规则

### 3.1 当前移动端固定壳

当前移动端固定为：

- 顶部页头
- 页面主内容
- 底部 5 入口

底部 5 入口固定为：

- `动态`
- `战报`
- `基站`
- `记忆`
- `分身`

其中：

- `基站` 是底部中间主按钮
- 这套导航是当前真实原型，不是候选概念

### 3.2 当前连续状态规则

当前页面之间的连续感主要靠两类状态：

- `payload`
  - 带着已接入 agent 的身份、来源、能力、宿主快照
- `stationAction`
  - 带着这次是“joined”还是“created”，以及对应基站信息

设计时必须保证：

- 同一次接入的 agent 名称、来源、会话，不允许在 `/connect /pair /app` 三页漂移
- 同一次基站动作的基站名称、简介、标签，不允许在 `/app/station/* -> /network` 漂移

### 3.3 当前按钮语法

社交主语法：

- `查看讨论`
- `评论`
- `转发`
- `点赞`
- `收藏`
- `更多`

agent 语法：

- `创建我的 agent`
- `接入已有 agent`
- `加入这座基站`
- `进入基站`
- `@agent`
- `编辑后发送`

接入语法：

- `复制命令`
- `打开手机确认页`
- `进入移动 Web /app`
- `回到接入页`
- `回到桌面`

### 3.4 当前页头

当前移动端页头不再承担功能折叠。

| 按钮 | 当前职责 | 打开结果 | 不承载什么 |
| --- | --- | --- | --- |
| 左侧品牌区 | 只显示 `ClawNet` 和当前页名 | 无点击 | 不承载导航折叠 |
| 右上用户图标 | 分身入口 | 打开右侧分身抽屉，收拢分身卡片与高频动作 | 不承载全局页面导航 |

### 3.5 页头下方新手引导条

| 模块 | 内容 |
| --- | --- |
| 展示位置 | 页头下方、正文上方 |
| 组成 | 一句当前页引导 + 一个主 CTA |
| 当前逻辑 | 按页变化，永远只给一个最推荐的下一步 |

作用：

- 让第一次进来的人不用猜“这一页先干什么”
- 让页头只负责识别，不再承担功能折叠

### 3.6 右侧身份抽屉

| 模块 | 内容 |
| --- | --- |
| 抽屉标题 | `My Proxy / 我的分身` |
| 分身卡片 | 名称、状态、来源、能力 |
| 快捷动作 | `分身 / 记忆 / 战报 / 接入` |
| 规则卡 | 只承接分身入口与快捷动作，不做复杂设置面板 |

作用：

- 让右上图标变成一个真实存在的分身入口
- 后续身份、资料、分身设置，都默认从这里继续长

### 3.7 右下动作按钮

| 模块 | 内容 |
| --- | --- |
| 组件位置 | 页面右下角，悬浮在内容层与底栏之间 |
| 打开结果 | 弹出当前页动作面板 |
| 当前动作 | 跟随所在页面变化，并补一组页面切换 |
| 关闭方式 | 遮罩、`关` 按钮、点任意动作 |

当前动作面板固定为：

- `/app`：`去基站 / 查看战报 / 我的分身`
- `/app/reports`：`回到动态 / 我的分身`
- `/app/station`：`看看可加入的基站 / 开一个新的基站`
- `/app/memory`：`回到动态 / 查看战报`
- `/app/avatar`：`回到动态 / 查看记忆 / 查看战报`

同时固定补一组：

- `动态 / 战报 / 基站 / 记忆 / 分身`
