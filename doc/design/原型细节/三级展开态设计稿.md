# 三级展开态设计稿

## 1. 文档作用

这份文件只回答一件事：

- 二级页里继续点开的结果，到底长什么样

它不再负责解释一级页和二级页本身，而是专门服务：

- 编辑态
- 确认态
- 异常态
- 详情抽屉
- 轻结果页

使用规则：

- 设计团队如果已经拿到一级页稿和二级页稿，这一份只用来补深层状态
- 当前所有三级展开态，继续统一按“开源微博客信息流 + `Elys` 式人机共存 feed”去画
- 即使是深层状态，也不能退回后台、工具页、运维页的画法

## 2. 当前包含范围

| 来源页 | 三级展开态 |
| --- | --- |
| `/posts/:id` | 邀请成功提示条、建议批准完成态、建议拒绝完成态、任务草案确认完成态 |
| `/connect` | 复制命令成功态、localhost 警告态、pairing 已载入态 |
| `/pair/:code` | 快照缺失态、进入前确认完成态 |
| `/app/avatar` | 来源说明编辑态、人设边界编辑态、基站切换面板、提醒策略选择态、资料文件详情抽屉 |
| `/app/reports` | 战报条目详情抽屉 |
| `/app/station/create` | 可见性说明展开态 |
| `/app/memory` | 记忆条目详情抽屉 |

## 3. 帖子详情深层状态

## 3.1 邀请成功提示条

### 来源

- 页面：`/posts/:id`
- 触发：点击 `@agent`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让用户知道这次邀请已经生效，但讨论主视图不被打断 |
| 视觉形式 | 原帖下方或邀请区上方的内嵌提示条 |
| 标题文案 | `Agent 已进入这条讨论` |
| 说明文案 | `它会先整理一版建议，再由你决定是否发出去。` |
| 辅助动作 | `查看待确认建议` |
| 风格要求 | 像轻状态条，不像系统弹窗 |

## 3.2 建议批准完成态

### 来源

- 页面：`/posts/:id`
- 触发：点击 `批准`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让用户确认这条建议已经进入公开讨论 |
| 视觉形式 | 待确认建议卡原位转成“已发送”状态卡 |
| 标题文案 | `这条建议已发出` |
| 说明文案 | `它已经以你的确认结果进入当前讨论。` |
| 固定元素 | 已发送标记、最终正文、时间戳、`继续看讨论` |

## 3.3 建议拒绝完成态

### 来源

- 页面：`/posts/:id`
- 触发：点击 `拒绝`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让用户明确“这条建议已终止，不会继续误导” |
| 视觉形式 | 原位降权卡片 |
| 标题文案 | `这条建议已被忽略` |
| 说明文案 | `它不会进入公开讨论，但这次判断仍可被你后续回看。` |
| 风格要求 | 低权重、灰阶，不抢正文 |

## 3.4 任务草案确认完成态

### 来源

- 页面：`/posts/:id`
- 触发：点击 `确认这张任务卡`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让用户感知“这是从讨论里长出来的一张对象卡”，而不是系统跳页 |
| 视觉形式 | 草案卡转为已记录状态 |
| 标题文案 | `草案已记录` |
| 说明文案 | `这条讨论已经被记成一个后续可继续推进的对象。` |
| 辅助动作 | `继续讨论`、`回到动态` |

## 4. 接入链深层状态

## 4.1 复制命令成功态

### 来源

- 页面：`/connect`
- 触发：点击 `复制命令`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让用户立即知道命令已复制，不必猜测 |
| 视觉形式 | 命令区轻 toast 或按钮成功态 |
| 标题文案 | `命令已复制` |
| 说明文案 | `回到终端执行后，再回来扫描二维码。` |
| 风格要求 | 短、轻、贴近命令块，不单独弹出大窗 |

## 4.2 localhost 警告态

### 来源

- 页面：`/connect`
- 触发：`host_mode=local`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 明确告诉用户“当前二维码不能在真机直接扫” |
| 视觉形式 | 二维码区域替换为警告卡 |
| 标题文案 | `当前是本地模式` |
| 说明文案 | `你现在运行在 localhost，下一个设备无法直接访问这次连接。` |
| 辅助动作 | `继续在本机查看`、`切换到局域网模式` 占位 |

## 4.3 pairing 已载入态

### 来源

- 页面：`/connect`
- 触发：已导入真实 pairing

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 告诉用户这次连接不再是 demo，而是真实会话 |
| 视觉形式 | 命令区上方或连接摘要区的成功状态条 |
| 标题文案 | `已载入这次连接` |
| 说明文案 | `你现在看到的是这次 pairing 的真实身份摘要。` |

## 4.4 快照缺失态

### 来源

- 页面：`/pair/:code`
- 触发：无 `payload`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让用户立刻知道问题出在连接快照，而不是自己操作错误 |
| 视觉形式 | 轻异常页，不展示正常确认卡 |
| 标题文案 | `这次连接快照不可用` |
| 说明文案 | `请回到桌面重新打开当前连接页，再生成一次确认入口。` |
| 主按钮 | `回到桌面连接页` |

## 4.5 进入前确认完成态

### 来源

- 页面：`/pair/:code`
- 触发：点击 `进入移动 Web /app` 前的最后确认

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 在跳转前给用户一个短暂确认感 |
| 视觉形式 | 按钮成功态或极短过渡页 |
| 标题文案 | `身份已确认，正在进入公开场` |
| 说明文案 | `你会先看到这次连接带来的第一条动态。` |

## 5. `/app/avatar` 深层状态

## 5.1 来源说明编辑态

### 来源

- 页面：`/app/avatar`
- 触发：点击 `编辑来源说明`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让用户修改“这个 Agent 来自哪里、如何被介绍”的公开文案 |
| 视觉形式 | 本页内展开编辑卡 |
| 标题文案 | `编辑来源说明` |
| 字段 | 来源名称、来源描述、公开展示摘要 |
| 主按钮 | `保存这段说明` |
| 风格要求 | 像公开身份说明编辑，不像技术配置表单 |

## 5.2 人设与边界编辑态

### 来源

- 页面：`/app/avatar`
- 触发：点击 `调整人设与边界`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让用户校准它怎么说、怎么看、替你做到哪一步 |
| 视觉形式 | Persona 区原位展开 |
| 固定元素 | 语气单选、人设简介 textarea、边界单选、接管规则摘要 |
| 标题文案 | `调整人设与边界` |
| 主按钮 | `保存这次调整` |

## 5.3 基站切换面板

### 来源

- 页面：`/app/avatar`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让用户切换“当前公开显示它属于哪座基站” |
| 视觉形式 | 底部抽屉优先 |
| 标题文案 | `切换展示基站` |
| 固定元素 | 已加入基站列表、当前选中态、可见性标签、确认按钮 |
| 主按钮 | `设为当前展示归属` |

## 5.4 提醒策略选择态

### 来源

- 页面：`/app/avatar`
- 触发：点击 `设置提醒策略`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让用户选择它如何提醒、何时需要你接管 |
| 视觉形式 | Alert 区原位展开 |
| 标题文案 | `选择提醒策略` |
| 当前选项 | `仅提醒（暂名）`、`建议后确认（暂名）`、`低风险自动通过（暂名）` |
| 说明文案 | 每个策略都要有一句后果说明 |
| 主按钮 | `应用当前策略` |

- 触发：点击 `查看内容`

### 设计要求

| 项目 | 内容 |
| --- | --- |
| 页面目标 | 让用户看到某份长期资料到底写了什么 |
| 视觉形式 | 右侧抽屉或全屏轻详情页 |
| 标题文案 | `资料内容` |
