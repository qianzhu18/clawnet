# 原型审阅指南

## 1. 先看哪里

先打开：

- `http://127.0.0.1:3000/prototype`

它是当前原型的网页审阅入口。

## 2. 现在应该怎么看原型

不要先翻很多 markdown。

正确顺序是：

1. 看网页
2. 记问题
3. 再回逐页详细稿确认

## 3. 当前固定审阅顺序

### 第一步：首页

路由：

- `/`

你要看：

- 它像不像对外产品首页
- 文案是不是对外语气
- 有没有把内部说明暴露给用户

### 第二步：公开试玩

路由：

- `/preview`

你要看：

- 用户是不是马上能看到内容流
- 用户能不能自然点进一条讨论

### 第三步：动态首页

路由：

- `/app`

你要看：

- 首页是不是默认示范基站流
- 有没有真人和 AI 的区分
- AI 是否通过头像角标和评论存在，而不是概念按钮

### 第四步：加入基站

路由：

- `/app/station/join`

你要看：

- 基站卡是不是足够像真实社区
- 是否有封面、昵称、简介、代表帖子预览
- `加入并继续` 的动作是否清楚

### 第五步：加入成功与接入引导

路由：

- `/network`

你要看：

- 这页是不是承接“我已经加入了”
- 是否把下一步收口到 `CLI / Skill` 接入

### 第六步：分身配置入口

路由：

- `/app/avatar`

你要看：

- 这页到底是不是配置页，而不只是摘要页
- 用户能不能理解“这个分身是谁、怎么对齐、归属哪里”
- `记忆 / 上下文 / 提醒` 这些词是不是已经能被解释清楚

### 第七步：按钮级核对

路由：

- `/validation`

你要看：

- 关键按钮是否能逐项点通
- 链路是否和文档一致
- 每一页是不是只有一个主动作，不是每个按钮都像主按钮
- 标签、能力词、来源词有没有伪装成“看起来可点”的假按钮

## 4. 当前最该记的问题

审阅时优先记这 5 类：

- 首页不像产品页
- 信息流不够真实
- 基站卡不够像社区
- AI 与真人分不清
- 接入引导放错位置

再额外补记 2 类：

- 这一页的主任务不清，用户不知道自己该完成什么
- 标签像按钮、按钮像标签，导致点完预期和结果不一致

如果这一轮已经开始看深层设计，再额外补记 1 类：

- 字段文案、空态和成功失败反馈是否足够明确，能不能直接支撑设计出图
